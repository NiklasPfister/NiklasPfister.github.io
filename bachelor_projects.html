Bachelor projects in statistics

This pages outlines thesis projects in statistics for the bachelor’s degree in mathematics or mathematics-economy at the University of Copenhagen. You may either pick one of the suggested topics or come up with your own project.

Info meeting: Monday, August 28, 3:15-4:00pm, in [TBA] at HCØ.

Formalities

The thesis is written during block 1 and block 2, 2023. The start date is September 4 and the thesis is handed in on January 12. There is a subsequent oral defense.

  • The thesis should be written in English.
  • It’s a 15 ECTS project and you should expect to write between 30 and 45 pages.
  • You should be signed up via Selvbetjeningen.
  • You will have to decide which project you will work on by August 31 (email: np@math.ku.dk)
  • You have to send a proposed title and description of your project by September 3 at 3pm (and you will get feedback).
  • You will have to fill out and submit the contract before September 4.

Details on supervision

The type of supervision will depend on whether there are sufficiently many students working on the same project as you. Overall, you should expect less supervision with projects on which very few students are working. So if you select such a project, you should feel comfortable working independently.

  • More than 5 students on same project:

    • During block 1: We will meet in group and you (the students) will present the background literature and theory, and we will have time for questions, both general questions as well as questions specific to what we are reading.

    • During block 2: We will meet individually. At these meetings you are expected to be shortly present your progress and have a set of prepared questions you would like to discuss.

  • Less than 5 students on same project:

    • During block 1: Short individual meetings. The goal of these meetings is to ensure that you spend time on relevant things, so you should prepare a detailed plan of what you plan to work on for each of these meetings.

    • During block 2: We will meet individually. At these meetings you are expected to be shortly present your progress and have a set of prepared questions you would like to discuss.

Individual meetings can be onsite or on zoom.

Prerequisites

The formal prerequisites are the courses Statistical Methods and Mathematical Statistics (or equivalent), but you are also expected to be interested in the following:

  • carry out data analysis and model validation on real data
  • implement models and/or data analyses (e.g. by writing R or Python scripts)
  • learn to use new software packages and functions
  • find relevant literature
  • independently read up on the background theory of the project
  • write a project that reflects theory as well as applications

Advice on writing

You are strongly encouraged to follow the recommendations outlined in the following writing guide.

As the project is applied it is important to ensure that your empirical results can be reproduced. For this it can be helpful to either work with R Markdown (and perhaps also Tidyverse) or using Jupyter notebooks, to keep track of the experiments you have performed. See also:

  • R for Data Science by Garrett Grolemund and Hadley Wickham
  • R Markdown: The Definitive Guide by Yihui Xie, J. J. Allaire, Garrett Grolemund

Project 2: Human gut microbiome - Distribution generalization

Microorganisms, such as bacteria, fungi and viruses interact in various ways with their surroundings. The human body is estimated to be a habitat for more than 10’000 different microbial species and they have been associated with various health outcomes such as cardiovascular disease, metabolic diseases, and obesity. Thanks to recent advances in gene sequencing technology the field of microbiome research, which measures the microorganisms based on gene sequencing, has made significant progress on understanding the links between the microbiome and its host.

One potential application of microbiome measurements is to use them as a diagnostic tool. For this one learns a predictive model that uses the microbiome to predict a specific phenotype that might be hard to measure in practice. An important problem with such predictive models is that they often do not generalize well to people with different characteristics than the participants in the training sample. In this project, you will investigate how to construct predictive models that are robust to distributional shifts (sometimes called distribution generalization). As a toy data set you can work with the publicily available ‘American Gut Project’ data, contains microbiome measurements and corresponding metadata (e.g., age, sex, BMI). A potential avenue to investigate might be to consider different types of variable selection procedures and analyze how they affect generalization performance.

The final report should include an overview of data as well as a short introduction to any of the methods that were used in the analysis. The main part of the project should then outline the discoveries including a clear description of how the different prediction models were compared. The literature list below provides several tools and methods that can be used in the analysis.

It is highly recommended to use Python for this project, but other software is allowed.

Data

For this project you will use data from the American Gut Project http://americangut.org. This is an initiative that enrolls regular people around the world (mainly USA and UK) and collects data from each participant (consisting of a gut microbiome sample and a questionnaire).

  • A preprocessed subset of data that is sufficient for this project is available at https://github.com/jacobbien/trac-reproducible.

Literature

  • American Gut: an Open Platform for Citizen Science Microbiome Research. Paper describing the American Gut Project.

  • https://github.com/knightlab-analyses/american-gut-analyses Github repository for American Gut Project, providing details on downloading full data (not required for this project but may be useful to get more information on it).

Some tools that might help in the analysis:

  • Supervised Learning and Model Analysis with Compositional Data. A method for a regression-based analysis that adjust for the compositional nature of the sequencing data. Method is available as a python package.

  • Regression models for compositional data: General log-contrast formulations, proximal optimization, and microbiome data applications. Extension of the log-contrast model to include sparsity penalty. Method is available as a python package.

  • Inferring Correlation Networks from Genomic Survey Data. A method to construct a graphical network on sequencing data. Method is available as a python package.
